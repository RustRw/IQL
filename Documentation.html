Преимущества

  1. Исключение синтаксических ошибок при составлении запроса

  2. Запросы INSERT и UPDATE в SQL, несмотря на схожесть операций, имеют абсолютно разный вид

  3. Упрощение работы с входными данными - не нужно использовать методы setString(), setInt() и другие методы ввода данных из PreparedStatment, достаточно просто ввести соотвествующие переменные

  4. Операция UPSERT, которая вставит, либо обновит запись если указан WHERE

  5. Генерация исключения, если DELETE или UPDATE выполняются без WHERE

  6. Возможность дополнительной обработки строк перед вставкой (при помощи специального функционального интерфейса)

  7. Подстветка синтаксиса при использовании IDE (чего, обычно, лишена прямая запись SQL-запросов)

  8. Автоматический парсинг даты из числа int, из строки String, либо из объекта Date

  9. Порядок команд не имеет значения, например GROUP BY можно указать после LIMIT, WHERE после ORDER BY, а JOIN перед SELECT

  10. Каждый запрос начинается с ввода таблиц, например для запроса SELECT сначала добавляются необходимые таблицы методом addTable(), тоже самое нужно сделать и для запросов UPDATE, INSERT, DELETE

  11. Может вернуть сгенерированный SQL-код, в отличие от PreparedStatement

  12. Упрощение многострочной вставки данных - не нужно конкатенировать запросы - достаточно просто вызывать метод insert() для каждой новой строки

Это функциональный интерфейс, который служит для обработки строк. Его метод:



String filter(String in);

Сортировка по полю row из активной таблицы в направлении type

Примеры использования

Простая выборка:

Connection con = DriverManager.getConnection("url","login","password");

IQL iql = new IQL(con);
iql.addTable("table1");
iql.select();
PreparedStatement st = iql.getStatement();

Этот же код можно записать следующим образом:

PreparedStatement st = new IQL(con).addTable("table1").select().getStatement();

Этот код создаст объекст типа PreparedStatement с SQL кодом:



SELECT * FROM `table1`

Условия:

Простое условие:

IQL iql = new IQL(con);
iql.addTable("table1");
iql.select();
iql.where("id %i", IQL.EQUAL, 11);
PreparedStatement st = iql.getStatement();



Сгенерированный код:



SELECT * FROM `table1` WHERE `table1`.`id` = 11

Сложное условие:

IQL iql = new IQL(con);
iql.addTable("table1");
iql.select();
iql.where("id %i", IQL.EQUAL, 11);
iql.

openBracket().
where("name %s", IQL.LIKE, "John%").

or().

where("name %s", IQL.LIKE, "Michael%").
closeBracket();
PreparedStatement st = iql.getStatement();

SQL:

SELECT * FROM `table1` WHERE `table1`.`id` = 11 AND (`table1`.`name` LIKE 'John%' OR `table1`.`name` LIKE 'Michael%')

Выборка определенных полей:

IQL iql = new IQL(con);
iql.addTable("table1");
iql.select("id", "name");
PreparedStatement st = iql.getStatement();


Сгенерирует следующий SQL-код:

SELECT `table1`.`id` AS `table1_id`, `table1`.`name` AS `table1_name` FROM `table1`



Как видно из примера - указание полей для метода select() приводит к указанию в запросе псевдонимов полей при помощи AS в виде имятаблицы_имяполя. Это сделано для того, чтобы имена столбцов не накладывались друг на друга при объединении таблиц. Для указания собственного псевдонима для поля его нужно ввести через пробел после самого имени поля:

iql.select("id id", "name name");

Например:

IQL iql = new IQL(con);
iql.addTable("table1");
iql.select("id id", "name username");
PreparedStatement st = iql.getStatement();

Приведет к созданию следующего кода:

SELECT `table1`.`id` AS `id`, `table1`.`name` AS `username` FROM `table1`



Объединение таблиц:

IQL iql = new IQL(con);
iql.addTable("users", "messages");
iql.join(1, "id", 2, "user_id");
iql.setTable(1).select("id user_id", "login login");
iql.setTable(2).select("id message_id", "date", "text text");
PreparedStatement st = iql.getStatement();

Сгенерирует следующий код:

SELECT

`users`.`id` AS `user_id`,

`users`.`login` AS `login`,

`messages`.`id` AS `message_id`,

`messages`.`date` AS `messages_date`,

`messages`.`text` AS `text`

FROM `users`

JOIN `messages` ON `users`.`id` = `messages`.`user_id`

Выборка из нескольких таблиц с LIMIT, GROUP BY и ORDER BY

IQL iql = new IQL(con);
iql.addTable("users","organisation_user","organisations");
//Номера таблиц 1 2 3
iql.join(1, "id", 2, "user_id");
iql.join(2, "organisation_id", 3, "id");
iql.setTable(1).select("login login").where("login %s", IQL.LIKE, "d0%");
iql.setTable(3).select("organisation_name orgname").where("type %i", "=", 1);
iql.groupBy("organisation_name", 3);
iql.orderBy("organisation_name", IQL.DESC, 3);
iql.limit(10);
PreparedStatement ps = iql.getStatement();

Сгенерирует следующий код:

SELECT

`users`.`login` AS `login`,

`organisations`.`organisation_name` AS `orgname`

FROM `users`

JOIN `organisation_user` ON `users`.`id` = `organisation_user`.`user_id`

JOIN `organisations` ON `organisation_user`.`organisation_id` = `organisations`.`id`

WHERE

`users`.`login` LIKE 'd0%' AND

`organisations`.`type` = 1

GROUP BY `organisations`.`organisation_name`

ORDER BY `organisations`.`organisation_name` DESC

LIMIT 0, 10

Создание таблиц:

Способ №1:

PreparedStatement st = new IQL(con).createTable("mytable", "name %s", "date %d", "desc %t", "price %i").getStatement();

Способ №2:

IQL iql = new IQL(con);
iql.createTable("mytable");
iql.addRow("name %s", "date %d", "desc %t", "price %i");
PreparedStatement st = iql.getStatement();

Способ №3:

IQL iql = new IQL(con);
iql.createTable("mytable");
iql.addRow("name %s");
iql.addRow("date %d");
iql.addRow("desc %t");
iql.addRow("price %i");
PreparedStatement st = iql.getStatement();

Приведет к генерации следующего SQL-кода:

CREATE TABLE `mytable`(`id` INTEGER PRIMARY KEY AUTO_INCREMENT, `name` VARCHAR(255), `date` INTEGER, `desc` TEXT, `price` INTEGER)

Вставка данных:

IQL iql = new IQL(con);
iql.addTable("mytable");
iql.setInsertRows("name %s", "register_date %d", "level %i");
iql.insert("User1", "17.05.2017", 4);
iql.insert("User2", "12.03.2016", 5);
Statement st = iql.getStatement();

Сгенерирует следующий код:

INSERT INTO `mytable`(`name`, `register_date`, `level`) VALUES ('User1', 1494968400, 4), ('User2', 1457730000, 5)

Следует отметить, что формат даты для %d, в случае получения объекта String для такого поля, по умолчанию установлен в dd.MM.yyyy

Обновление данных:

IQL iql = new IQL(con);
iql.addTable("organisations");
iql.setUpdateRows("name %s", "address %s");
iql.update("New orgname", "New address");
iql.whereId(112);
PreparedStatement ps = iql.getStatement();



Сгенерирует следующий код:

UPDATE `organisations` SET `name` = 'New orgname', `address` = 'New address' WHERE `organisations`.`id` = 112



Стоит отметить, что методы getStatement() и getSQL() для этого запроса могут выбросить исключение InsecureOperationException (RuntimeException), если не вызван ни один из методов where() или whereId()



Операция UPSERT

Эта операция не является прямой SQL-операцией, а является всего лишь оберткой над UPDATE и INSERT

Например, переделанный вариант предыдущего примера:

IQL iql = new IQL(con);
iql.addTable("organisations");
iql.setUpsertRows("name %s", "address %s");
iql.upsert("New orgname", "New address");
iql.whereId(112);
PreparedStatement ps = iql.getStatement();

сгенерирует тот же код:

UPDATE `organisations` SET `name` = 'New orgname', `address` = 'New address' WHERE `organisations`.`id` = 112

Однако, если не произведено вызова метода where(), либо whereId(), например:

IQL iql = new IQL(con);
iql.addTable("organisations");
iql.setUpsertRows("name %s", "address %s");
iql.upsert("New orgname", "New address");
PreparedStatement ps = iql.getStatement();

То будет сгенерирован SQL код для вставки данных:

INSERT INTO `organisations`(`name`, `address`) VALUES ('New orgname', 'New address')

Удаление данных

Удаление по id:

IQL iql = new IQL(con);
iql.addTable("users");
iql.delete(12);
PreparedStatement ps = iql.getStatement();

Сгенерирует следующий код:

DELETE FROM `users` WHERE `users`.`id` = 12

Удаление по произвольному полю:

IQL iql = new IQL(con);
iql.addTable("users");
iql.delete();
iql.where("login_date %d", IQL.LESS, "18.05.2015");
PreparedStatement ps = iql.getStatement();

Сгенерирует следующий код:

DELETE FROM `users` WHERE `users`.`login_date` < 1431896400

Стоит отметить, что методы getStatement() и getSQL() для этого запроса могут выбросить исключение InsecureOperationException (RuntimeException), если не вызван ни один из методов where() или whereId()
